## 治理与非单一化

代码由项目维护者迁移或删除；修改任一映射必须 supersede 本 ADR，并更新 deterministic trace。键盘、触控与 standard-mapped gamepad 最终进入同一 aggregate action；没有某一种设备的输入可获得更高权重。movement 与 signal 都能贡献 meaningful fact，不形成唯一最优输入。所有计数只用于 guard/可追溯事实，不合并为 score、rank、评价或结局标签。

浏览器 RAF、Gamepad API 与 haptics 的设备差异仍可能隐藏低刷新率、后台节流和无震动设备；因此 wall-gap 输入采用 fail-closed policy，且不声称未经实机矩阵验证的硬件一致性。

## 行为契约

- seed / RNG domain：Run 接受 caller-resolved uint32 encounter seed；URL 未提供 seed 时才生成 entropy，一旦显式提供则必须是十进制 uint32，否则 fail closed，不回退随机数。不发明 V4 未提供的 difficulty salt；每个 combat pattern 仍只使用一个 Mulberry32 stream。
- AWAKENING duration：选择 8000ms（V4 `[6000,10000]` 的固定中点），但只有 `meaningfulInputCount >= 2` 才交接；guard 未满足时保持安静，不越过 narrative authority。
- meaningful input：movement magnitude 从 `<= 0.15` 到 `> 0.15`，或 signal 从 inactive 到 active，记一次 rising fact；同一 authority tick 两者同时 rise 仍只记一个 aggregate fact，避免单指触控获得双倍权重；持续 held 不重复计数。
- signal / Flower：inactive/active binary signal 分别映射到 V4 AWAKENING `0.30–0.50` 带的端点 `0.3/0.5`；Focus、Gaze、Override 的优先级和 cap 仍由 V4 `FlowerIntensityResolver` 决定。Focus 是 `0.35` 上限，不会把 `0.3` 抬高成新 target；不生成新 waveform。
- gaze：`CanonicalRunSession` 只消费调用方提供的 `{skyEyeVisible, pitchDegrees, alignment}` device-neutral sample，不从鼠标、渲染、摇杆或摄像头自行推断。V4 GazeMachine 在连续 60 `tick120` 合格后提交 clamp，失配 54 `tick120` 后提交 release；事件在精确 deadline tick 进入 run-scoped ordered bus。clamp 同 tick 先提交 gaze 事实，再由 Flower resolver 强制 `0.1`。gaze 与玩家身体生命状态独立；death/respawn 不能把仍合格的 caller sample 改写成 release。
- gaze source precedence：runtime state-machine manifest 的 `>=45° / alignment >= 0.72 / 500ms` 高于 narrative raw guard 的 `>45° / directness >= 0.55`；本 adapter 不混合阈值。`event-projections-v4` 中 `gaze.clamp.commit → gaze.clamp.begin` 与 release identity projection、FIRST_EYE transition shape 均被启动时钉住，drift 时 fail closed。
- input unlock：AWAKENING 只让 move + signal 写入；Focus 在 First Eye 才生效。player 处于 dead/respawning/run-ended 时，player-sourced movement、Focus、binary signal 与 Override 被门控，只在精确 respawn-complete tick 恢复；独立 gaze 事实与其已获得的 Flower 优先级不被身体死亡改写。当前 fragment 不进入 `LOCAL_RESISTANCE_AVAILABLE`，因此不会让 First Eye 的 evidence 提前打开局部 Override。
- pause：pause wall time全部丢弃；pause 内边沿不带回历史 tick。若 pause 到达时 clock 仍有 pre-pause backlog，其原 authority control/Override edge 样本必须保留到 backlog 排空，不能用中性输入重写历史。追上 wall head 后才将当前 aggregate held state 与 authority-held state 比较，并在下一 tick 合成至多一个 edge。
- RAF/backlog：一个 RAF 的 elapsed interval 只使用上一个已存在 sample；当前 poll 只能写未来 tick。超过 1024 boundary 的 backlog 先用旧 sample 排空；wall gap 中的瞬时边沿不回填历史，追上 wall head 后只对 held state 做一次 reconciliation。
- aim lead：负 lead 读取精确 crossed `tick120` 历史；V4 未定义在线未来采样，正 lead 暂使用最后一段 authority movement 的线性外推并 clamp 到 viewport。
- combat omissions：graze radius、通用 projectile damage、archetype→pool class、First Eye room/difficulty 由 immutable adapter policy 记录；当前选择为 `18px`、`1`、`bullet.micro.notch_e→micro`、`bullet.micro.dash→micro`、`INFORMATION/EASY`。
- run combat ownership：`CanonicalRunCombatState` 持有一个 `CanonicalEventBus`、player position、`PlayerDamageAuthority`、`EvidenceAuthority`、`GrazeEvidenceAuthority` 与 `DirectionalOverrideAuthority`；kernel 只持有 pattern/RNG/schedule/projectile pool/runtime projectiles。shared kernel 必须提供未使用过的 Unicode-scalar `occurrenceId`，entity/occurrence scope 为 `combat:<UTF-8 bytes>:<occurrenceId>:<patternId>`。当前只允许一个 spawning occurrence；parallel、multi-pool fanout 与 aggregate budget 不在本扩展内。
- exact-tick lease：state 构造时对 exact-brand/prototype bus 取得不可逆 lease，并把 start tick 标记为 closed。之后只接受 `closedThroughTick120 + 1` 的 plain own-data draft；ambient `flush()`、第二 owner、future/stale tick、accessor、sparse/metadata/virtual array 与 reentrant mutation 均拒绝。`advanceTick()` 只准备当前 tick，允许 coordinator 添加同 tick 事实；`flushTick()` 即使没有 event 也推进 closed-through，一次成功 close 后才 finalize occurrence release。
- prepared receipt：event bus 在完整合并 append 受理后，为每个原始 draft-group identity 签发一个 opaque receipt。Boss、laser 与 player after-state 必须在 owner/bus/revision 仍有效时消费其 exact receipt；空 group 也以 array identity 区分，wrong-bus、wrong-group、same-revision sibling、stale、replay 或无 append 都拒绝。该 receipt 只绑定本次 append 与预计算 after-state，不能回滚任意 sibling mutation。
