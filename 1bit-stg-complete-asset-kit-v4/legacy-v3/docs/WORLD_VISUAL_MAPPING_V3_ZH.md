# 1bit 世界观到 v3 STG 视觉系统的映射

本文描述最终包中的视觉语义；机器可读资源边界以 `manifests/v3/asset-manifest-v3.json` 为准，文件完整性以根目录 `checksums-sha256.txt` 为准。

## 1. 不改变的核心

《1bit》的核心不是复古黑白，也不是“玩家打败权威”。它是一台记录身体如何应对治理的机器：玩家在表达、暴露、压抑、误读与短暂反抗之间移动，运行结束后得到一份非评判性的行为快照。

v3 把原有探索空间转译成传统竖版 STG，但不改变这五个关系：

1. **系统 / Ink**：绝对规则、碰撞边界、读取与遮蔽。
2. **自我 / Paper**：花核、信号、短暂可见的观察者。
3. **摩擦 / Dither or residue**：系统与自我接触后留下的错位、磨损和烧屏。
4. **天空之眼 / Gaze**：权威不需要追逐玩家；被看见本身已经改变玩家状态。
5. **状态快照 / Snapshot**：记录发生过什么，不把玩家排成善恶、强弱或结局等级。

v3 的八色不是对 1-bit 概念的背离。`SYSTEM_INK / SELF_PAPER` 仍是隐喻骨架，四个房间色只说明“当前是哪一种读取协议”，`OVERRIDE_MAGENTA` 只说明 scar 正在被写入。

## 2. 从第一人称探索到 STG

原世界观中的动作不能原样搬成弹幕按钮，必须保留它们的权力关系。

| 原始动作／状态 | STG 转译 | 视觉后果 |
|---|---|---|
| 行走 | 在弹幕治理坐标中移动 | 路径、停顿、折返与擦边成为 trace |
| 调亮花 | Z 射击／持续表达 | 火力上升，同时光强、刷新与被读取概率上升 |
| 主动调暗 | Shift Focus | 壳体缓慢收拢，路径与射线变窄；这是主动“入神” |
| 抬头看 Eye | 进入 Eye 的读取／夹紧条件 | 120ms 内突然压暗，有冻结与恢复延迟；这是被夺走节奏 |
| 在 crack／seam 中停留 | 穿越两套弹幕协议的交界 | 两侧读取同时作用；不提供默认安全答案 |
| Override | X 消耗擦边证据，局部改写 | 定向 tear、local Void、局部碰撞关闭、scar 写入、沉积 |
| 运行结束 | State Snapshot | 确定性指纹、事实指标、观察句与下一局条件 |

Focus 和 Gaze 都可能降低可见亮度，但过程完全不同：前者连续、可逆、由玩家发起；后者突然、强迫、恢复延迟。把它们做成同一个“缩小动画”会抹掉世界观。

Override 也不是全屏 Bomb。它只在一个有限方向和有限区域让规则失效；规则恢复后 scar 仍在。反抗的意义来自留下坐标，不来自把屏幕清空。

## 3. Internet Void：行为大于图标

互联网负空间不是浏览器窗口、准星、404、矩阵字符雨或电路板。那些都是内容的图标；本项目要显示的是被系统忽略的行为。

因此 Void 在 v3 中通过以下事实出现：

- 敌弹朝运动方向缺一口，说明“此处禁止占据”也有可被绕开的边缘；
- 玩家停顿比走过的路线更厚，系统遗漏的犹豫被保留下来；
- INFO 的 packet 反复尝试却没有稳定中心；
- A/B 系统各自成立，只有交集成为可学习的碰撞真值；
- 数字壳已删除，物质残留仍多停一拍；
- ghost 只复演一次，而 scar 跨局保留；
- witness 不是背景摆件，它会因真实 scar、路径终点或玩家亮度而转向。

如果一个 sprite 只能被描述为“眼睛图标”“网络图标”或“日式纹章”，它还没有抵达行为层。

## 4. 数字／物质双螺旋

每条关键链都要同时回答两个问题：规则在数字层怎样变化？材料在现实层留下什么？

| 系统 | 数字轨 | 物质轨 |
|---|---|---|
| 玩家受击 | collision off、壳体删除 | 碎片与沉积多停一拍 |
| 敌人死亡 | AI 与命中资格先关闭 | 不完整余料成为 residue |
| Override | 局部规则关闭、bbox 暴露 | scar 与粉尘沉降 |
| INFO | packet loss、retry、stale route | 被反复压印又中断的热敏纸带 |
| FORCED | 同一 seed 被两套规则执行 | 同一模具脱出的两块不等量余料 |
| IN-BETWEEN | 独立 A/B 时钟与稳定交集 | 两张不同纤维方向的板材叠压 |
| POLARIZED | hard threshold 与强制镜像 | 红色切片、黑断面、骨白芯材 |
| Snapshot | 行为被压成可查询元数据 | 指纹、scar、burn-in 成为可见存档 |

数字轨不应总是“干净”、物质轨也不应总是“真实”。两者都是同一治理与生产系统的不同表面。

## 5. 四个精神状态房间

v3 的机器 ID 与原文名称有一组对应关系：

| 原世界观 | v3 资源 ID | 不可删除的行为 |
|---|---|---|
| `INFO_OVERFLOW` | `INFORMATION` / `information` | 更多信号不等于更多理解；刷新、丢包、重试、烧屏 |
| `FORCED_ALIGNMENT` | `FORCED_CHOICE` / `forced_choice` | 同一输入被迫成为左右两种结果；seam 不是安全答案 |
| `IN_BETWEEN` | `IN_BETWEEN` / `in_between` | 同一行为被两套协议相反读取；碰撞使用稳定交集 |
| `POLARIZED` | `POLARIZED` / `polarized` | 没有灰区与黄昏；只有 scar 能破坏系统镜像 |

### INFORMATION

- 视觉没有稳定中心；不同步的纵向束反复刷新。
- 玩家越亮，刷新和注意压力越高，但不会得到更多回答。
- 旧波次可以留下无碰撞 burn-in，干扰记忆但不造成伤害。
- 背景 trace 应来自真实旧路径，而不是随机噪点。

### FORCED CHOICE

- 左右共享 seed，结构同源，损耗与结果不同。
- seam 必须有活动、桥段或门控，不能被读成永久安全通道。
- 完全站到任一侧可以更稳定，但代价是另一种读取被压低；系统不宣布哪侧正确。
- 两个 Boss 不是“秩序阵营 vs 混乱阵营”，而是单一规则与双重主权的两种压力。

### IN-BETWEEN

- 系统 A 与 B 使用独立节拍、方向和 sprite offset。
- 视觉可以不完全重合，危险真值必须稳定、公平、可学习。
- 默认碰撞是 `A.contains(point) && B.contains(point)`；抖动副本不能修改冻结的玩法形状。
- 不把混合身份转成漂亮的双月徽章；错位必须影响读取和路径。

### POLARIZED

- 只有硬边、矩形、开／关与无情的离散节拍。
- 禁用 gray、渐变、抖动与黄昏 tween。
- 系统层可以镜像；玩家历史 trace 必须保持非镜像。
- Absolute Reader / Sky Eye 是唯一允许使用巨大破损同心眼的对象。
- Override 的结果不是永久胜利，只是一次局部、有限的规则破口。

## 6. 八个 Boss 是八种治理拓扑

Boss 的差异不来自颜色或徽章，而来自负空间如何组织玩家路线。

| Boss | 规则如何占据空间 |
|---|---|
| Absent Receiver | 所有输入抵达偏心空槽后消失；没有回应 |
| Unanswering Feed | 流束持续更新，却绕开一个永不填满的缺口 |
| One Sun, One Rule | 单一横尺决定所有阴影与校正方向 |
| Two Claims | 两块不等宽的权利区共享边界，互不相同 |
| Misreader | 两个相差一像素／一拍的窗口无法同时满足 |
| Misregistered Twin Moons | 两个未闭合的新月制造错位走廊 |
| No Dusk | 时间墙只有开与关，拒绝过渡 |
| Absolute Reader | 巨大阅读之眼试图总览一切，圆周被玩家 scar 打断 |

前七个 Boss 若再次变成同心圆、四角准星或径向曼陀罗，就会退化成同一种权威图标。只有最后一个 Boss 可以占用这一视觉特权。

## 7. 跨局记忆

跨局不是结算页彩蛋，而是下一局真实状态：

```text
scarRehydrate
→ ghostReplayOnce
→ witnessTurn
→ returnInput
```

- **scar**：保存房间、归一化坐标、方向与 run ID；跨局持久。
- **ghost**：回放足以辨认停顿、折返与擦边的稀疏路径；无碰撞、无奖励，只播一次。
- **witness**：朝 scar、ghost 终点或玩家出生点转向；不是预烘焙装饰。
- **burn-in**：刚被读取的高对比画面逐像素熄灭；它是记忆残留，不是 VHS 滤镜。
- **snapshot**：同一 run data 必须确定性生成同一指纹，并供失败、Continue、暂停 mini preview 与下一局使用。

数字壳先消失、材料多停一拍，是这套世界观最小的时间单位。若死亡、删除、天气和跨局都同时淡出，双螺旋就只剩说明文字。

## 8. “抽象日式”从哪里来

本项目的日式感不是文化图腾，也不是复制某款游戏。它来自对像素、停顿与留白的组织：

### 8.1 像素经济

- `360×640` 的掌机式小画幅；
- 一个轮廓只保留改变判断所需的像素；
- 小而明确的 silhouette，不靠纹理堆满面积；
- 同一对象只在必要位置移动一像素；
- 可爱与不安不由角色表情制造，而由节拍突然翻转。

### 8.2 时间中的“间”

- Focus：收拢后停 `240ms`，再用一像素确认；
- Eye：突然夹紧后冻结，恢复故意迟到；
- 数字删除之后，材料继续存在；
- 菜单确认前后有短停顿，不用按钮发光代替反馈；
- 跨局先让 scar 出现，再让 ghost 走完，最后才归还输入。

停顿不是缺帧，而是动作的一部分。它必须写进 duration、hold 与事件时序。

### 8.3 构图中的留白

- Gameplay HUD 只贴四边，中部留给判断；
- Boss integrity 是一把断尺，不是完整容器；
- 标题 mark 偏心、不闭合；
- 负空间不居中，不自动变成漂亮圆孔；
- 中文主信息、英文小注，层级克制，不把页面填满。

### 8.4 Deadpan warmth

文案描述事实，不评判人格：

- 可以：“你在被完全看见之前改写了规则。”
- 不可以：“你是一位勇敢的反抗者。”

温度来自系统仍愿意记住一个动作，不来自吉祥物、萌化表情或怀旧引用。

### 8.5 明确禁用

- 鸟居、家纹、日轮、樱花、浮世绘波、假名、书法、和服；
- 魔法阵、曼陀罗、徽章、符文、宝石、药水；
- 可识别的 Mother、Touhou 或其他作品角色、UI 与原始配色；
- 电路板城市、Matrix 字符雨、浏览器窗口墙、VHS/RGB glitch；
- bloom、柔光、玻璃、金属倒角、渐变和亚像素噪点。

## 9. UI 是元数据，不是外壳

UI 只显示会改变判断、动作或记忆的量：

- Light：玩家主动表达的强度；
- Gaze：系统施加的读取压力；
- Override：擦边证据是否足以进行局部改写；
- Room：当前协议；
- Boss integrity：制度装置的连续性，而不是生物血量；
- Snapshot：事实指标、行为标签和下一局条件。

Light 与 Gaze 不能合并为一条能量。Continue 必须先说明上一局将以什么形式进入下一局，再要求确认。失败页不显示 `GAME OVER`，状态快照不显示 Rank、善恶值和“最佳结局”。

## 10. 视觉审查时的十个问题

1. 去掉颜色后，玩家、敌弹、敌人、拾取物和 memory 是否仍能分开？
2. 这个形状说明了哪一种行为，而不只是说明了哪一种题材？
3. Void 是否有方向，还是一个漂亮的居中孔？
4. 数字规则改变后，材料是否多留下了一拍？
5. Focus 与 Gaze 的主动／强迫差异能否只看节拍读出？
6. 四个房间在换成同一色后，运动规则是否仍不同？
7. 前七个 Boss 是否又滑回眼、圆环、准星与径向徽记？
8. Reduced Motion 是否保留了相同因果与事件，而不是删掉过程？
9. State Snapshot 是否只陈述可证实行为？
10. 如果删除一个像素或一层，信息是否仍然成立？若成立，就继续做减法。
